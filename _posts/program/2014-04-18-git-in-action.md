    本篇文章里，我们结合一个测试项目，介绍一下git的基本命令和git的最佳实践。git为分布式开发而生，很多人对git的了解还停留在简单的代码托管工具上。其实，git是一个很强大的版本管理系统。
    很多人还是在master上开发自己的代码，然后直接提交到master上，当然，对于只有一人开发的项目而言，这样没有什么问题。如果只是这样使用git，那实在是太大材小用了。
    git可以很方便地管理代码版本，多人开发时的优势尤其明显。
